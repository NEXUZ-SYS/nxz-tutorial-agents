# Checkpoint: Revisao do Draft do Briefing

Apresente o briefing completo ao usuario para revisao.

## Apresentar ao Usuario

1. O briefing completo em Markdown
2. Checklist de qualidade:
   - [ ] Metadata completa
   - [ ] Secoes adequadas ao publico/nivel
   - [ ] Exemplos praticos presentes
   - [ ] Cross-references com produtos relacionados
   - [ ] Glossario incluido
   - [ ] Linguagem adequada ao publico

## Perguntas

### 1. O briefing atende ao objetivo definido?

- Sim, esta excelente — aprovar como esta
- Precisa de ajustes menores — vou descrever o que mudar
- Precisa de reescrita significativa — ha problemas estruturais
- Quero adicionar/remover secoes especificas
