# Checkpoint: Revisao dos Dados Coletados

Antes de escrever o briefing, apresente ao usuario um resumo dos dados coletados:

## Apresentar ao Usuario

1. **Analise de contextos existentes**: Resumo das lacunas encontradas
2. **Respostas da entrevista**: Pontos-chave sobre o produto
3. **Exploracao do produto**: Funcionalidades validadas e screenshots capturados
4. **Plano do briefing**: Secoes que serao incluidas com base no publico/nivel

## Perguntas

### 1. Os dados coletados estao corretos e completos?

- Sim, pode prosseguir com a escrita do briefing
- Ha informacoes incorretas que preciso corrigir
- Faltam informacoes importantes — quero adicionar mais dados
- Quero ajustar o escopo (publico, nivel ou objetivo)
