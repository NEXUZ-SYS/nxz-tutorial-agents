# Checkpoint: Configuracao do Briefing de Contexto

Apresente ao usuario as opcoes para configurar o briefing a ser criado.

## Perguntas

### 1. Qual produto Nexuz deseja documentar?

- **NXZ ERP** — Sistema completo de gestao (Odoo 12): compras, vendas, PDV, estoque, financeiro, faturamento
- **NXZ Go** — App/totem de autoatendimento: pedidos personalizados, pagamentos rapidos
- **NXZ KDS** — Kitchen Display System: gestao de pedidos na cozinha
- **NXZ Delivery** — Centralizador de pedidos de delivery: iFood, Rappi, NXZ Go
- **NXZ Pay Go** — Solucao de pagamento mobile

O usuario pode escolher um produto ou descrever um modulo especifico dentro de um produto (ex: "NXZ ERP - Modulo PDV").

### 2. Qual o publico-alvo deste briefing?

- **Tech (Dev/QA/Ops)** — Linguagem tecnica, APIs, stack, models, endpoints
- **Business** — Linguagem de negocios, fluxos, ROI, processos
- **Sales** — Linguagem comercial, proposta de valor, diferenciais competitivos
- **Marketing** — Linguagem de marca, beneficios, UX, copy

### 3. Qual o nivel organizacional?

- **Estrategico** — Visao executiva: roadmap, mercado, KPIs, posicionamento
- **Tatico** — Gestao: processos, integracoes, metricas operacionais
- **Operacional** — Uso diario: passo a passo, funcionalidades, fluxos de trabalho

### 4. Qual o objetivo especifico deste briefing?

Exemplos:
- Alimentar um agente de IA de suporte ao cliente
- Base de conhecimento para treinamento de equipe de vendas
- Contexto para geracao de conteudo de marketing
- Documentacao tecnica para integracao entre sistemas
- Onboarding de novos colaboradores
- Contexto para chatbot de FAQ

O usuario descreve livremente o objetivo.
