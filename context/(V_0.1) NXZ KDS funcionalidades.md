# Funcionalidades — KDS (Kitchen Display System)

> Lista completa de funcionalidades do display de cozinha.

## Navegação

O KDS usa navegação por abas (Bottom Tabs):

| Aba           | Descrição                                 | Badge                          |
| ------------- | ----------------------------------------- | ------------------------------ |
| **Home**      | Pedidos ativos/em produção                | Contador de pedidos pendentes  |
| **Delivery**  | Pedidos de delivery (iFood/Open Delivery) | Contador de entregas pendentes |
| **Histórico** | Pedidos finalizados                       | Contador de finalizados        |

A aba Delivery só aparece quando `company.ifood_id` está configurado.

## Home — Pedidos Ativos

- **Cards de pedidos** em FlatList, ordenados por:
  1. Pedidos iFood não confirmados (prioridade máxima — aguardando aceite)
  2. Pedidos por data de criação (`created_at`)
- **Filtro de agendados** — Toggle para exibir/ocultar pedidos iFood com timing `SCHEDULED`, com badge de contagem
- **Pedidos "pending finish"** — Pedidos com todos os itens prontos, aguardando finalização do operador
- Exclui automaticamente pedidos finalizados e de delivery

## Card de Pedido (Order)

Cada card exibe:

- **Header**: Número do pedido, canal de origem (POS, iFood, Open Delivery, Keeta, 99Food, VCQPad), horário, tipo (dine-in/takeaway)
- **Itens**: Lista de produtos com quantidades e stage individual
- **Detalhes de delivery**: Endereço, dados do entregador (quando aplicável)
- **Info de agendamento**: Horário previsto (iFood scheduled)
- **Tempo do pedido**: Desde a criação

### Ações por Pedido

- **Avançar stage** — Avança itens individualmente ou todos de uma vez
- **Imprimir** — Ticket, sticker ou DANFE
- **Aceitar/Rejeitar** — Para pedidos iFood/draft com modais de confirmação
- **Disputar** — Para pedidos iFood, com campo de motivo
- **Auto-print** — Impressão automática na chegada de novo pedido (se `autoPrint` configurado)

## Stages de Produção

Cada item de produção (`IProduction`) possui um stage:

| Stage        | Significado         |
| ------------ | ------------------- |
| `''` (vazio) | Aguardando          |
| `progress`   | Em preparação       |
| `done`       | Pronto              |
| `confirmed`  | Confirmado/expedido |

O operador avança stages individualmente por item. O cron `ProductionItem` gerencia transições automáticas.

## Filtro de Categorias

Permite que diferentes estações de trabalho (grill, bar, sobremesas) vejam apenas itens relevantes.

- **Ativação**: `kdsConfig.use_category_filter`
- **Categorias**: Lista de IDs configuráveis por KDS
- **Ação para itens fora da categoria**:

| Ação         | Comportamento                                  |
| ------------ | ---------------------------------------------- |
| `no_warning` | Exibe normalmente                              |
| `warning`    | Exibe com indicador visual de alerta           |
| `hide`       | Oculta itens fora da categoria                 |
| `block`      | Bloqueia interação com itens fora da categoria |

O filtro é desabilitado (`block`) nas abas Delivery e Histórico para visibilidade completa.

## Integração iFood

- **Aceitar/Rejeitar** pedidos com modal de confirmação
- **Disputar** pedidos com campo de motivo
- **Status do merchant** — Indicador no header (aberto/fechado/alerta) com modal de detalhes
- **Auto-aceite** — Configurável via `AutoAcceptIfoodOrders`
- **Pedidos agendados** — Exibidos separadamente com filtro toggle
- **Cron `IFoodStatus`** — Sincroniza status do merchant periodicamente

## Integração Open Delivery

Suporte a pedidos de plataformas Open Delivery:

- Keeta
- 99Food
- VCQPad
- Outros via padrão Open Delivery

## Delivery

- Aba dedicada para pedidos de delivery
- Exibe pedidos onde `order.delivery === true && !order.finished`
- Filtro de categorias desabilitado para visibilidade total dos itens

## Histórico

- Pedidos finalizados (`isOrderDoneForKds` ou `order.finished`)
- Ordenados por data decrescente (mais recentes primeiro)
- Filtro de categorias desabilitado
- Máximo de 100 pedidos retidos (limpeza automática via `clearHistory()`)

## Header

- Nome do usuário e empresa
- Logo da loja
- **Botão "PDV"** — Troca para modo PDV (se `user.mode === 'BOTH'`), com modal de confirmação
- **Indicador iFood** — Status da loja (aberto/fechado) com modal de detalhes
- **Filtro de agendados** — Toggle com badge de contagem
- **Relógio** — Atualizado a cada 20 segundos

## Pedidos Draft e Comandas

- Pedidos salvos como draft (`isDraft`) ou comanda (`isOpenTab`) são visíveis no KDS
- Permite preparação antecipada mesmo sem pagamento confirmado
- Gestão de retomar/cancelar via tela OrderManagement (modo BOTH)

## Real-time e Sincronização

### Firebase Realtime Database

- Estrutura: `/erp-<seller>/<company_id>/orders[<branch>]/<orderId>`
- Listener `onValue` para atualizações instantâneas
- `onDisconnect` marca POS como offline automaticamente
- Campos `stage` e `note` do KDS são preservados durante atualizações vindas do PDV
- Health check: Write + Read + Delete com timeout de 5s cada

### Supabase (alternativa)

- Tabela `orders` particionada por coluna `channel`
- Postgres change events (INSERT/UPDATE/DELETE) via realtime channel
- Callbacks registrados por UUID

### OrderManager (Singleton)

- Subscreve ao Firebase/Supabase via `Database.listener()`
- Emite eventos `update` via EventEmitter
- Componentes React subscrevem via `OrderManager.on('update', callback)`

## Background Jobs (Crons)

| Cron           | Frequência   | Função                                 |
| -------------- | ------------ | -------------------------------------- |
| ExcludeOrder   | Configurável | Remove pedidos antigos/finalizados     |
| IFoodStatus    | 30h          | Sincroniza status do merchant iFood    |
| ProductionItem | Configurável | Gerencia transição de stages           |
| ClearCache     | 24h          | Remove cache expirado                  |
| Device         | —            | Health check e registro do dispositivo |
| NotifyErrors   | —            | Pipeline de notificação de erros       |
| Upgrade        | —            | Verifica atualizações OTA              |

## Impressão

- **Ticket** — Resumo do pedido para cozinha
- **Sticker** — Etiqueta de identificação
- **DANFE** — Nota fiscal (quando aplicável)
- **Auto-print** — Impressão automática na chegada de novo pedido
- Mesmos drivers V1/V2/V3 do PDV

## Comunicação Remota

Ações recebidas via WebSocket:

- `Restart` — Reinicia o app
- `ClearCache` — Limpa cache do dispositivo
- `ClearCrons` — Reseta agendamento de crons
- `Debug` — Ativa/desativa modo debug
- `Status` — Reporta status do dispositivo
- `SendCFe` — Re-emissão de CFe SAT remota (busca no Odoo, envia ao SAT, retorna resultado)
