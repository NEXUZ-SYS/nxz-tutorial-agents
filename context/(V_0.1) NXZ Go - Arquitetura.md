# NxzGo — Moober Self Checkout

> Gerado em 2026-03-16 por análise do codebase (v5.13.1).

## O que é o NxzGo

O **NxzGo** é uma plataforma Android completa para operação de restaurantes e food service. Um único aplicativo React Native que se adapta a quatro modos de operação — **PDV**, **Totem**, **KDS** e **Smart POS** — cobrindo desde o registro do pedido pelo atendente ou cliente até a preparação na cozinha e expedição.

O NxzGo é o front-end de operação do ecossistema Nexuz. Ele não funciona isoladamente: depende do **ERP Odoo** como backend central (catálogo, sessões POS, vendas, fiscal, configurações) e de um **banco de dados real-time** (Firebase ou Supabase) para sincronizar pedidos entre dispositivos.

## Problema que Resolve

Restaurantes precisam de múltiplos sistemas para operar: caixa, autoatendimento, display de cozinha, maquininhas de pagamento e integrações com iFood. Cada um desses sistemas normalmente é um software diferente, de fornecedores diferentes, com integrações frágeis entre eles.

O NxzGo unifica tudo em um único app configurável. Um tablet no balcão vira PDV. Um tablet na entrada vira Totem. Um tablet na cozinha vira KDS. Uma maquininha Cielo/Stone/PagBank vira terminal de pedido completo. Todos conectados em tempo real, compartilhando catálogo, pedidos e configurações.

## Como os Modos se Conectam

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  PDV (Caixa) │     │    Totem     │     │  Smart POS   │
│  Atendente   │     │   Cliente    │     │  Maquininha  │
└──────┬───────┘     └──────┬───────┘     └──────┬───────┘
       │                    │                     │
       └────────────────────┼─────────────────────┘
                            │
                    ┌───────▼────────┐
                    │  Firebase /    │
                    │  Supabase      │
                    │  (real-time)   │
                    └───────┬────────┘
                            │
              ┌─────────────┼─────────────┐
              │             │             │
       ┌──────▼──────┐  ┌──▼───┐  ┌──────▼──────┐
       │  KDS Grill   │  │ KDS  │  │ KDS Drinks  │
       │  (filtro:    │  │ Bar  │  │  (filtro:   │
       │   hambúrger) │  │      │  │   bebidas)  │
       └─────────────┘  └──────┘  └─────────────┘
              │             │             │
              └─────────────┼─────────────┘
                            │
                    ┌───────▼────────┐
                    │   Odoo ERP     │
                    │  (backend)     │
                    └────────────────┘
```

1. **PDV, Totem ou Smart POS** criam o pedido e processam o pagamento
2. O pedido é salvo no **Firebase/Supabase** em tempo real
3. **KDS** recebe o pedido instantaneamente e exibe para a cozinha
4. Múltiplos KDS podem filtrar por categoria (grill, bar, sobremesas)
5. Tudo é sincronizado com o **Odoo ERP** para controle financeiro e fiscal

Além dos pedidos internos, o KDS também recebe pedidos de **iFood** e **Open Delivery** (Keeta, 99Food, VCQPad).

## Modos de Operação

| Modo | Config | Uso | Dispositivo Típico |
| --- | --- | --- | --- |
| **PDV (Caixa)** | `configs.type = 'caixa'` | Frente de caixa operada por atendente | Tablet Android |
| **Totem** | `configs.type = 'totem'` | Autoatendimento pelo consumidor final | Tablet 10"+ em suporte |
| **KDS** | `user.mode = 'KDS'` | Display de cozinha para produção | Tablet em braço articulado |
| **Smart POS** | `flavor: cielo/stone/pagbank/msitef` | Terminal completo na maquininha | Maquininha Android |

O modo `BOTH` (`user.mode = 'BOTH'`) permite alternar entre PDV e KDS no mesmo dispositivo.

PDV, Totem e Smart POS compartilham o mesmo fluxo de pedido e pagamento. As diferenças estão documentadas no [comparativo](./comparativo.md). O KDS tem interface e funcionalidades próprias.

## Para Quem É

- **Atendentes de caixa** — Registram pedidos e processam pagamentos (PDV)
- **Consumidores finais** — Fazem pedidos em totens de autoatendimento (Totem)
- **Equipe de cozinha** — Visualizam e gerenciam produção de pedidos (KDS)
- **Operadores de delivery** — Aceitam e gerenciam pedidos iFood/Open Delivery (KDS)
- **Atendentes com maquininha** — Pedido + pagamento + impressão em um único dispositivo (Smart POS)
- **Gerentes** — Controlam disponibilidade de produtos, configurações e monitoramento

## Principais Capacidades

- **Multi-provedor de pagamento** — Dinheiro, Sumup, Cielo, PagBank, Stone, Pagarme, PIX, com integração via API, Bluetooth e SDK nativo
- **Nota fiscal brasileira** — NFC-e (modelo 65) e SAT/CF-e (modelo 59) com emissão automática
- **White label** — Cores, logos, textos, screensavers e comportamentos customizáveis por estabelecimento
- **Offline-first** — Pedidos são cacheados localmente e sincronizados quando a conexão retorna
- **Real-time** — Pedidos aparecem no KDS instantaneamente via Firebase/Supabase
- **Multi-dispositivo** — Vários PDVs, Totems e KDS operando simultaneamente no mesmo estabelecimento
- **iFood e Open Delivery** — Aceite, rejeição e gerenciamento de pedidos de delivery
- **Impressão térmica** — Suporte a impressoras Bluetooth, rede, USB e nativas de Smart POS
